# Resume Reference (Tech/Data 2026)

## What recruiters want quickly
1. Role-fit evidence in first screen
2. Quantified outcomes and ownership
3. Reliability + delivery signals (tests, CI/CD, incident reduction)
4. Architecture judgment (cost/perf trade-offs)
5. Career progression and scope

## High-value bullet pattern
Use: **Accomplished X as measured by Y by doing Z**

Example:
- Reduced batch runtime by 73% (30m → 8m) by migrating full reload ETL to CDC merge upserts.

## ATS guidance
- Standard headings (Summary, Experience, Projects, Skills, Education)
- Parsable layout; avoid decorative structures
- Keywords in context, not pasted lists
- Consistent dates and title formatting

## Data/AI engineer signal areas
- Software craftsmanship: modular code, tests, CI/CD
- Semantic layer + governed metrics
- FinOps and architecture tradeoffs
- Orchestration, data quality, observability

## Bullet scoring rubric (0–2 each)
- Action clarity
- Business impact
- Metric specificity
- Technical depth
- JD relevance
- Concision

Target: 9+ / 12 for key bullets.

## Follow-up metric prompts
- Baseline and end state?
- Scale (rows/day, users, dollars, latency)?
- Ownership scope (solo/team/cross-functional)?
- Measured business outcome?
